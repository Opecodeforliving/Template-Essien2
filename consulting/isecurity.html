Security Risk Assessment

We provide security review following best-practice approach whereby a security assurance provider carries out a security snapshot view. Another security provider carries out remediation Thereafter the security assurance provider re-assesses security controls.

Thus essentially:

  • snapshot security review
  • Remediation – security control implementation
  • Post-implementation security review
  • Residual risk

The second stage (post implementation) is a mirror of the first stage focused on residual risk. We also carry out security remediation task as an independent body as the intermediary in a security review process. The Security Review process in this phase can be divided into three tasks, namely a risk assessment, Information Security Policy review and Information Architecture review and update. The risk assessment can in turn be divided into three sub-tasks, namely ICT business processes risk assessment, vulnerability assessment and penetration testing.
